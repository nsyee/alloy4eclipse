Alternatives to "static" and appended facts

In File System Lesson I, we wrote 
  // There exists a root
  one sig Root extends Dir {
  }{
    no parent
  }
Equivalently, we could have accomplished the same effect by writing two additional facts: 
  //Root is a kind of directory
  sig Root extends Dir {}

  //there is exactly one Root directory
  //(same effect as making Root "static")
  fact oneRoot {
    one Root
  }

  //Root has no parent
  //(same effect as the appended fact}
  fact noParentOfRoot
    no Root.parent
  }

We could also combine those two fact statements into one. Notice that when more than one constraint appears in a fact, they are separated by commas. 

  //Root is a kind of directory
  sig Root extends Dir {}

  //there is exactly one Root directory
  //Root has no parent
  fact oneRoot {
    one Root,
    no Root.parent
  }

We could also reword the second constraint in the fact (although there is no particular reason to do so) as follows: 

  //Root is a kind of directory
  sig Root extends Dir {}

  //there is exactly one Root directory
  //Root has no parent
  fact oneRoot {
    one Root,
    all r:Root | no r.parent
  }

this rewriting works because "." gets mapped onto the elements of a set it is applies to.

Both of these syntatic shorthands can make a model more concise and clearer, but are not always appropriate. When and how you use them is a matter of style.
